1. Inputs

  • Profile 1 - The triangular profile of refractive index in the optical fiber core.
  • Profile 2 - The gradient profile of refractive index in the optical fiber core.
  • Profile 3 - The step profile of refractive index in the optical fiber core.
  • Profile 4 - The step profile of refractive index in the optical fiber core with depressive cladding.
  • Profile 5 - The step profile of refractive index in the optical fiber core with a depressive ring around the core.
  • Germanium - A molecular concentration of germanium admixture in the optical fiber core [M%].
  • Fluoride - A molecular concentration of fluoride admixture in the optical fiber cladding [M%].
  • Wavelength - A wavelength for the optical fiber concerned [um].
  • a - A greater than zero value determining range of the 'n1' refractive index [um].
  • b - A Non-negative value determining range of the 'n2' xor 'n3' refractive index [um].
  • c - A Non-negative value determining range of the 'n3' refractive index [um].
  • q - A power which determine smoothness of the gradient profile.
  • 2D points - A number of points making up the 2D plot.
  • 3D points - A number of points making up the 3D plot.
  • 2D plot - Determining whether to create the 2D plot.
  • 3D plot - Determining whether to create the 3D plot.

2. Outputs

  • n1 - The refractive index in the optical fiber core.
  • n2 - The refractive index in the optical fiber cladding.
  • n3 - The refractive index in the optical fiber depressive cladding xor ring.
  • V1 - The Verdet constant in the optical fiber core [rad/(T*m)].
  • V2 - The Verdet constant in the optical fiber cladding [rad/(T*m)].
  • V3 - The Verdet constant in the optical fiber depressive cladding xor ring [rad/(T*m)].
  • Cutoff wavelength - Cutoff wavelength [um].

3. URL parameters

When the application detects arguments in the URL, the user will be asked to load them. Only valid values will be loaded. This is one of the ways to easily save and share a specific input configuration.


Available parameters:

namevalue
profile1 / 2 / 3 / 4 / 5
germaniumdecimal
fluoridedecimal
wavelengthdecimal
adecimal
bdecimal
cdecimal
qdecimal
points2Dinteger
points3Dinteger
plot2Dtrue / false
plot3Dtrue / false

How to use?

Example application URL: example.com/modeler.html

1. append one query char ? at the end of URL
2. append name=value pair at the end of URL
3. connect next pair by using the ampersand character &


Example:

example.com/modeler.html?profile=5&germanium=6.34&fluoride=1&a=5.7&points2D=100&plot3D=false
